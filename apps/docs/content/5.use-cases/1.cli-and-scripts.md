evlog works as a drop-in logger for any TypeScript script or CLI tool — no framework required. Use `log.*` for step-by-step terminal output and a single `createLogger` block to capture the whole run as one wide event you can ship to your drain.

Set up evlog in my CLI tool or one-shot TypeScript script.

- Install evlog: pnpm add evlog
- Call initLogger({ env: { service: 'my-cli' }, drain }) once at startup
- Wrap the run in a single createLogger({ task, ... }) block to emit one wide event with totals + duration
- Use log.info / log.warn / log.debug for step-by-step terminal progress
- Parse --verbose / --quiet flags into minLevel: 'debug' | 'info' | 'error'
- Throw createError({ message, why, fix, link }) for actionable failures
- ALWAYS await drain.flush() in a finally block before process.exit

Docs: https://www.evlog.dev/use-cases/cli-and-scripts
Standalone setup: https://www.evlog.dev/frameworks/standalone

::

## Pattern

1. `initLogger` once at startup with a meaningful `service` name.
2. Use `log.*` calls inside the script for human-readable progress (`log.info`, `log.warn`, `log.debug`).
3. Wrap the run in one `createLogger` block to emit a single summary event with totals, duration, and outcome.
4. `await drain.flush()` before `process.exit` so you don't lose buffered events.

## Minimal example

```typescript [scripts/migrate-users.ts]
import { initLogger, log, createLogger, createError } from 'evlog'
import type { DrainContext } from 'evlog'
import { createAxiomDrain } from 'evlog/axiom'
import { createDrainPipeline } from 'evlog/pipeline'

const pipeline = createDrainPipeline<DrainContext>({
  batch: { size: 25, intervalMs: 2000 },
  retry: { maxAttempts: 3 },
})
const drain = pipeline(createAxiomDrain())

initLogger({
  env: { service: 'migrate-users', environment: process.env.NODE_ENV ?? 'development' },
  drain,
})

const run = createLogger({ task: 'user-migration', startedAt: new Date().toISOString() })

try {
  log.info('migrate', 'Loading legacy users')
  const users = await db.query('SELECT * FROM legacy_users')
  run.set({ found: users.length })

  let migrated = 0
  let failed = 0
  for (const user of users) {
    try {
      await newDb.upsert({ id: user.id, email: user.email, plan: user.plan })
      migrated++
      log.debug({ event: 'user_migrated', id: user.id })
    } catch (err) {
      failed++
      log.warn({ event: 'user_failed', id: user.id, reason: (err as Error).message })
    }
  }

  run.set({ migrated, failed, status: failed === 0 ? 'complete' : 'partial' })
  log.info('migrate', `Done: ${migrated} migrated, ${failed} failed`)
} catch (err) {
  run.error(err as Error)
  log.error({ event: 'migration_aborted' })
  process.exitCode = 1
} finally {
  run.emit()
  await drain.flush()
}
```

```bash [Output (dev)]
14:58:12 [migrate] Loading legacy users
14:58:14 [migrate] Done: 1248 migrated, 2 failed
14:58:14 INFO [migrate-users] in 1842ms
  ├─ task: user-migration
  ├─ found: 1250
  ├─ migrated: 1248
  ├─ failed: 2
  └─ status: partial
```

## Verbose / quiet flags

CLI tools usually want a `--verbose` flag. Pass `minLevel` based on the parsed flag:

```typescript [scripts/cli.ts]
import { initLogger } from 'evlog'

const verbose = process.argv.includes('--verbose')
const quiet = process.argv.includes('--quiet')

initLogger({
  env: { service: 'my-cli' },
  minLevel: quiet ? 'error' : verbose ? 'debug' : 'info',
})
```

::callout{icon="i-lucide-info" color="info"}
`minLevel` is set once at `initLogger` time (see [Honest gaps](/getting-started/vs-other-loggers#honest-gaps-today)). For CLIs that toggle verbosity mid-run, parse flags before any logging call.
::

## Structured errors that print well

Throw `createError` for anything the user might need to act on. The CLI can catch it and pretty-print without losing the `why` / `fix` data:

```typescript [scripts/deploy.ts]
import { createError, parseError, log } from 'evlog'

try {
  await deploy()
} catch (err) {
  const { message, why, fix, link } = parseError(err)
  log.error({ event: 'deploy_failed', message, why, fix, link })

  console.error(`\n× ${message}`)
  if (why) console.error(`  Why: ${why}`)
  if (fix) console.error(`  Fix: ${fix}`)
  if (link) console.error(`  Docs: ${link}`)
  process.exit(1)
}
```

## Don't forget `drain.flush()`

Drains are async and batched — exiting the process before the batch is sent loses events. Always `await drain.flush()` in your `finally` block.

::callout{icon="i-lucide-triangle-alert" color="warning"}
On a long-lived CLI (e.g. a watch mode), you don't need to flush manually — the pipeline batches on its own. Flush only on shutdown / `process.exit`.
::
