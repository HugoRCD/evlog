`@evlog/eval` is an AI eval runner built on evlog primitives. It follows the same `dataset + task + scorers` pattern as Evalite and Braintrust, but every case is captured as a structured wide event — meaning it drains anywhere evlog drains.

## Install

```bash [pnpm]
pnpm add @evlog/eval
```

The AI SDK is optional — only required if you use `llmJudge` or `ai.wrap()` inside tasks:

```bash [pnpm]
pnpm add ai
```

## Quick Start

```typescript [evals/summarization.eval.ts]
import { createEval, exactMatch, llmJudge } from '@evlog/eval'
import { maxSteps, noToolLoop } from '@evlog/eval/ai-scorers'
import { jsonlDrain } from '@evlog/eval/drains'
import { generateText } from 'ai'

const myEval = createEval({
  name: 'summarization-quality',
  dataset: [
    { id: 'c1', input: 'Long article about TypeScript...', expected: 'TypeScript is a typed superset of JavaScript.' },
    { id: 'c2', input: 'Nitro is a server toolkit...', expected: 'Nitro builds universal server apps.' },
  ],
  task: async (input, { ai }) => {
    const model = ai.wrap('anthropic/claude-haiku-4')
    const { text } = await generateText({
      model,
      prompt: `Summarize in one sentence: ${input}`,
    })
    return text
  },
  scorers: [
    exactMatch,
    llmJudge({ model: 'anthropic/claude-haiku-4', threshold: 0.7 }),
  ],
  drain: jsonlDrain(),
  concurrency: 5,
  threshold: 0.8,
})

const results = await myEval.run()
console.log(`Pass rate: ${(results.passRate * 100).toFixed(1)}%`)
console.log(`Cost: $${results.estimatedCost?.toFixed(4) ?? 'n/a'}`)

if (!results.passed) process.exit(1)
```

Run it directly with Node or tsx:

```bash
npx tsx evals/summarization.eval.ts
```

## What Each Case Emits

Every case produces a standard evlog `WideEvent` with two extra fields — `eval` and `ai` — drained through the normal pipeline:

```json [Wide Event]
{
  "timestamp": "2026-05-12T22:00:00.000Z",
  "level": "info",
  "service": "eval",
  "environment": "development",
  "eval": {
    "name": "summarization-quality",
    "caseId": "c1",
    "input": "Long article about TypeScript...",
    "output": "TypeScript is a typed superset of JavaScript.",
    "expected": "TypeScript is a typed superset of JavaScript.",
    "scores": { "exact-match": true, "llm-judge": 0.92 },
    "passed": true,
    "durationMs": 342
  },
  "ai": {
    "calls": 1,
    "model": "claude-haiku-4",
    "provider": "anthropic",
    "inputTokens": 450,
    "outputTokens": 87,
    "cacheReadTokens": 120,
    "estimatedCost": 0.0018,
    "tokensPerSecond": 156,
    "finishReason": "stop"
  }
}
```

The `ai.*` fields are captured automatically — no extra instrumentation. Every task receives a pre-wired `createAILogger(log)` in its context.

## EvalResults

`run()` returns aggregated results:

```typescript
const results = await myEval.run()

results.name          // 'summarization-quality'
results.passed        // true when passRate >= threshold
results.passRate      // 0.84
results.total         // 50
results.passing       // 42
results.avgScores     // { 'exact-match': 0.62, 'llm-judge': 0.91 }

// Aggregated from ai.* fields across all cases
results.totalInputTokens    // 22500
results.totalOutputTokens   // 4350
results.totalCacheReadTokens // 6000
results.estimatedCost       // 0.089

// Timing percentiles
results.p50DurationMs // 340
results.p95DurationMs // 1200
```

## Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `name` | `string` | required | Eval name. Written to `eval.name` on every wide event. |
| `dataset` | `EvalCase[] \| AsyncIterable \| () => Promise<EvalCase[]>` | required | Test cases. |
| `task` | `(input, ctx) => output` | required | The function under evaluation. |
| `scorers` | `Scorer[]` | `[]` | Per-case scorers. See [Scorers](/use-cases/eval/scorers). |
| `summaryScorers` | `SummaryScorer[]` | `[]` | Post-run scorers that see all results. |
| `drain` | `DrainFn \| DrainFn[]` | `undefined` | One or more drains. See [Drains](/use-cases/eval/drains). |
| `concurrency` | `number` | `5` | Max cases running in parallel. |
| `trialCount` | `number` | `1` | Run each case N times. Scores are averaged across trials. |
| `threshold` | `number` | `undefined` | Minimum pass rate for `results.passed` to be `true`. |
| `timeout` | `number` | `undefined` | Per-case timeout in ms. |
| `aiOptions` | `AILoggerOptions` | `undefined` | Forwarded to `createAILogger`. Set `cost` here for `estimatedCost`. |
| `service` | `string` | `'eval'` | Service name on every wide event. |

## Task Context

The `task` function receives `(input, ctx)` where `ctx` contains:

```typescript
interface EvalTaskContext {
  log: RequestLogger  // standalone evlog logger for this case
  ai: AILogger        // pre-wired createAILogger(log) — call ai.wrap(model)
  case: EvalCase      // the current case
  trialIndex: number  // 0-based trial index (when trialCount > 1)
}
```

Both `log` and `ai` are decoupled from any HTTP context — they work the same in a script as in a server handler.

## Trial Runs

Set `trialCount` to run each case multiple times and measure variance:

```typescript
const myEval = createEval({
  name: 'agent-consistency',
  dataset: [...],
  task: async (input, { ai }) => { /* ... */ },
  scorers: [exactMatch],
  trialCount: 5, // run each case 5 times
})
```

Numeric scores are averaged across trials. Boolean scores use majority vote (3/5 passing → `true`). Individual trial data is available in `results.cases[i].trials`.

## CI Integration

Use `threshold` to fail the process when quality drops:

```typescript
const results = await myEval.run()

if (!results.passed) {
  console.error(`Eval failed: ${(results.passRate * 100).toFixed(1)}% pass rate (threshold: ${threshold * 100}%)`)
  process.exit(1)
}
```
