Scorers evaluate the output of a task. Each scorer receives a `ScorerContext` and returns a score — a number 0-1, a boolean, or `{ score, metadata }` for richer output.

## Built-in Output Scorers

Import from `@evlog/eval`:

```typescript
import { exactMatch, contains, matches, notEmpty, lengthBetween } from '@evlog/eval'
```

| Scorer | Passes when | Import |
|--------|-------------|--------|
| `exactMatch` | `output === expected` | `@evlog/eval` |
| `contains(str)` | output includes the substring | `@evlog/eval` |
| `matches(regex)` | output matches the pattern | `@evlog/eval` |
| `notEmpty` | output is not empty after trim | `@evlog/eval` |
| `lengthBetween(min, max)` | output length is in range | `@evlog/eval` |

```typescript
import { createEval, exactMatch, contains, matches } from '@evlog/eval'

const myEval = createEval({
  name: 'output-checks',
  dataset: [{ id: 'c1', input: 'What is TypeScript?', expected: 'TypeScript is a typed superset of JavaScript.' }],
  task: async (input, { ai }) => {
    const { text } = await generateText({ model: ai.wrap('anthropic/claude-haiku-4'), prompt: input })
    return text
  },
  scorers: [
    exactMatch,
    contains('TypeScript'),
    matches(/typed/i),
    lengthBetween(10, 500),
  ],
})
```

## LLM Judge

Use a language model to score output quality when there is no single correct answer:

```typescript
import { llmJudge } from '@evlog/eval'

const myEval = createEval({
  name: 'quality-check',
  dataset: [...],
  task: async (input, { ai }) => { /* ... */ },
  scorers: [
    llmJudge({
      model: 'anthropic/claude-haiku-4', // use a cheap model for judging
      threshold: 0.7,
    }),
  ],
})
```

The judge receives a structured prompt with `input`, `expected`, and `output` and returns a score 0-1. Use a small, cheap model (Haiku, Flash) rather than the same model you are evaluating.

### Custom prompt

Override the default prompt to give the judge more context:

```typescript
llmJudge({
  model: 'anthropic/claude-haiku-4',
  threshold: 0.7,
  prompt: ({ input, output, expected }) =>
    `You are a technical writer reviewing a documentation summary.
Score this summary from 0.0 (unusable) to 1.0 (publication-ready).

Original: ${input}
Expected tone: ${expected}
Actual: ${output}

Reply with a single number between 0.0 and 1.0.`,
})
```

### autoevals compatibility

Scorers from [`autoevals`](https://github.com/braintrustdata/autoevals) work directly — they follow the same 0-1 numeric shape:

```typescript
import { Levenshtein } from 'autoevals'
import { createEval } from '@evlog/eval'

const myEval = createEval({
  name: 'string-similarity',
  dataset: [...],
  task: async (input) => myApp(input),
  scorers: [Levenshtein],
})
```

## defineScorer

Create named, reusable scorers with `defineScorer`:

```typescript
import { defineScorer } from '@evlog/eval'

const startsWithCapital = defineScorer({
  name: 'starts-with-capital',
  score: ({ output }) => /^[A-Z]/.test(String(output)),
})

const wordCount = defineScorer({
  name: 'word-count-ok',
  score: ({ output }) => {
    const count = String(output).split(/\s+/).filter(Boolean).length
    return { score: count >= 5 && count <= 100, metadata: { wordCount: count } }
  },
})
```

The `score` function receives a `ScorerContext`:

```typescript
interface ScorerContext<TInput, TOutput> {
  input: TInput
  output: TOutput
  expected?: TOutput
  event: Readonly<WideEvent>  // access event.ai.* for AI telemetry
}
```

## Score Types

| Return value | How it's interpreted |
|---|---|
| `true` | Passes |
| `false` | Fails |
| `0.85` | Passes when `score >= threshold` (default 0.5) |
| `{ score: 0.85, metadata: { ... } }` | Score with extra data attached to the wide event |

Set a custom `threshold` on the scorer:

```typescript
defineScorer({
  name: 'quality',
  threshold: 0.8, // require 0.8+ to pass
  score: ({ output }) => computeQuality(output),
})
```

## Summary Scorers

Summary scorers run once after all cases are complete and receive the full result set:

```typescript
import { createEval } from '@evlog/eval'

const myEval = createEval({
  name: 'regression-check',
  dataset: [...],
  task: async (input) => myApp(input),
  scorers: [exactMatch],
  summaryScorers: [
    {
      name: 'majority-pass',
      score: (cases) => cases.filter(c => c.passed).length > cases.length / 2,
    },
    {
      name: 'no-catastrophic-failures',
      score: (cases) => {
        const failed = cases.filter(c => !c.passed)
        // fail the suite if any case scored 0
        return !failed.some(c => Object.values(c.scores).some(s => s === 0 || s === false))
      },
    },
  ],
})

const results = await myEval.run()
results.summaryScores // { 'majority-pass': true, 'no-catastrophic-failures': true }
```

Summary scores appear in `results.summaryScores` but do not affect `results.passed` — they are informational aggregates.
