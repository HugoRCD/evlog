Every eval case produces a `WideEvent` that flows through the same drain interface as the rest of evlog. This means every existing evlog adapter — Axiom, Datadog, OTLP, Better Stack, PostHog — works as an eval drain with no changes.

The `drain` option accepts a single drain function or an array. Multiple drains run in parallel.

## jsonlDrain — Local Output

`jsonlDrain` writes each case as a NDJSON line to a local file. The file is created (along with parent directories) if it does not exist.

```typescript
import { jsonlDrain } from '@evlog/eval/drains'

// Auto-path: .evlog/evals/{evalName}.jsonl
drain: jsonlDrain()

// Custom path
drain: jsonlDrain({ path: './results/summarization.jsonl' })
```

The output is git-diffable — one line per case makes it easy to compare results across branches or commits.

## braintrustDrain — Braintrust Experiments

`braintrustDrain` creates a Braintrust experiment and inserts each case row via the REST API. No Braintrust SDK is required — it uses native `fetch` only.

```typescript
import { braintrustDrain } from '@evlog/eval/drains'

drain: braintrustDrain({
  apiKey: process.env.BRAINTRUST_API_KEY!,
  projectName: 'my-app',
  experimentName: 'v1.2-summarizer',  // auto-generated if omitted
})
```

The drain maps:

| evlog field | Braintrust field |
|---|---|
| `event.eval.caseId` | row `id` |
| `event.eval.input` | `input` |
| `event.eval.output` | `output` |
| `event.eval.expected` | `expected` |
| `event.eval.scores` | `scores` (boolean → `1`/`0`) |
| `event.ai.*` | `metadata.ai` (tokens, cost, model, steps, toolCalls) |

The experiment is created lazily on the first drain call and reused for all subsequent cases in the run.

::tip
Pass a stable `experimentName` to accumulate results across multiple partial runs. Omit it to get a fresh timestamped experiment each time.
::

## evlog Adapters

Every evlog adapter works as an eval drain — the `DrainContext` interface is identical. Import them from the `evlog` package:

::code-group
```typescript [Axiom]
import { axiomDrain } from 'evlog/axiom'

drain: axiomDrain({
  token: process.env.AXIOM_TOKEN!,
  dataset: 'evals',
})
```

```typescript [Datadog]
import { datadogDrain } from 'evlog/datadog'

drain: datadogDrain({
  apiKey: process.env.DD_API_KEY!,
  service: 'my-evals',
})
```

```typescript [OTLP]
import { otlpDrain } from 'evlog/otlp'

drain: otlpDrain({
  endpoint: process.env.OTLP_ENDPOINT!,
})
```

```typescript [Better Stack]
import { betterStackDrain } from 'evlog/better-stack'

drain: betterStackDrain({
  sourceToken: process.env.BETTER_STACK_TOKEN!,
})
```
::

This gives eval results the same retention, search, and alerting as your production events — in the same dashboard.

## Multiple Drains

Pass an array to drain to multiple destinations in parallel:

```typescript
import { braintrustDrain, jsonlDrain } from '@evlog/eval/drains'
import { axiomDrain } from 'evlog/axiom'

drain: [
  braintrustDrain({ apiKey: process.env.BRAINTRUST_API_KEY!, projectName: 'my-app' }),
  axiomDrain({ token: process.env.AXIOM_TOKEN!, dataset: 'evals' }),
  jsonlDrain(),
]
```

Drain failures are logged to `console.error` and do not interrupt the eval run — a single broken drain does not stop other drains or the aggregation step.

## Per-run Override

Override the drain for a single run without modifying the eval definition:

```typescript
const myEval = createEval({
  name: 'summarization',
  drain: jsonlDrain(), // default for local dev
  // ...
})

// In CI: use Braintrust instead
await myEval.run({
  drain: braintrustDrain({ apiKey: process.env.BRAINTRUST_API_KEY!, projectName: 'ci' }),
})
```

## Custom Drain

Any function matching `(ctx: DrainContext) => void | Promise<void>` works as a drain:

```typescript
drain: async ({ event }) => {
  await db.insert('eval_results', {
    evalName: event.eval?.name,
    caseId: event.eval?.caseId,
    passed: event.eval?.passed,
    scores: event.eval?.scores,
    inputTokens: (event.ai as Record<string, number>)?.inputTokens,
  })
}
```

## Disable Drain

Omit the `drain` option to run evals locally without draining anywhere — results are still returned by `run()`:

```typescript
const results = await createEval({
  name: 'quick-check',
  dataset: [...],
  task: async (input) => myApp(input),
  scorers: [exactMatch],
  // no drain — results only in memory
}).run()
```
