A **source** is anything that creates wide events and feeds them into the [drain pipeline](/core-concepts/lifecycle). The built-in sources cover the HTTP frameworks evlog ships integrations for ([Frameworks overview](/frameworks/overview)), the [browser client](/logging/client-logging), and integrations with specific products ([AI SDK](/logging/ai-sdk/overview), [Better Auth](/logging/better-auth/overview)).

You write a custom source when:

- You want to support a new HTTP runtime that doesn't have a `evlog/<framework>` package yet
- You build a non-HTTP source (a queue worker, a CLI driver, a cron job) and want it to log wide events the same way
- You wrap evlog into your own internal framework

## Going further

Once your source feeds events into the pipeline, the rest of the chain is the same as for any built-in source — apply [enrichers](/build-on-top/pipeline/custom-enrichers), [drains](/build-on-top/sinks/custom-drains), or just [observe the flow](/build-on-top/observers).

If your integration is meant to be reused across projects, [package it as an npm library](/build-on-top/shared/integration-as-package).
