An **enricher** runs on every emitted event before it reaches drains. It's the right tool when you want a field on every event without touching every call site — geo, user agent, trace context, deploy id, tenant id…

Add a custom enricher so a field appears on every wide event automatically.

- Import `defineEnricher` from `evlog/toolkit` and return either a plain object of fields or a function `(event, ctx) => fields`
- Read from `ctx.headers`, `ctx.env`, `ctx.request` — never from `globalThis` request state
- Keep it pure and fast: no awaitable I/O on the hot path; cache anything expensive at module scope
- Wrap the body in try/catch and return `undefined` on failure — enrichers must never bring down a request
- Wire it on my framework: Nuxt/Nitro `evlog:enrich` hook, Hono/Express/Fastify/Elysia `{ enrichers: [...] }` middleware option, or `initLogger({ enrichers: [...] })` for standalone
- For multi-hook features (enrich + drain side-effect, etc.), use `definePlugin` instead

Docs: https://www.evlog.dev/build-on-top/custom-enrichers

## When to reach for plugins instead

If your feature mixes enrichment with other hooks (e.g. enrich + tail-sample + side-effect on drain), use a [plugin](/build-on-top/plugins) instead — one cohesive object covering several lifecycle points.
