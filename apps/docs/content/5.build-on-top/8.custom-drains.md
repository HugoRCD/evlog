A **drain** is a function that receives wide events and ships them somewhere terminal: an HTTP API, a message queue, a database, a webhook. evlog ships built-in drains for popular providers ([Adapters overview](/adapters/overview)). When you need a destination that isn't covered, write your own.

Ship wide events to a backend that doesn't have a built-in evlog adapter.

- For HTTP backends, use `defineHttpDrain({ name, resolve, encode })` from `evlog/toolkit` — it handles timeouts, retries, and adds the `User-Agent: evlog/<version>` + `X-Evlog-Source: <name>` identity headers automatically
- For non-HTTP transports (queue, DB, native SDK, raw socket), use `defineDrain({ name, send })` and implement `send(events)` myself
- Resolve config lazily inside `resolve()` so env vars and `runtimeConfig` are read at request time, not at module load
- Encode batched events into the destination's wire format inside `encode(events, config)` — return `{ url, headers, body }`
- Wire it on my framework like any other drain (Nuxt: `evlog.adapters.custom`, Next/standalone: `initLogger({ drain })`, others: pass to the framework middleware)

Docs: https://www.evlog.dev/build-on-top/custom-drains

## A 5-minute example — internal Loki drain

```ts
import { defineHttpDrain } from 'evlog/toolkit'

export function createLokiDrain(overrides?: { url?: string; token?: string }) {
  return defineHttpDrain<{ url: string; token: string }>({
    name: 'loki',
    resolve: () => ({
      url: overrides?.url ?? process.env.LOKI_URL!,
      token: overrides?.token ?? process.env.LOKI_TOKEN!,
    }),
    encode: (events, config) => ({
      url: `${config.url}/loki/api/v1/push`,
      headers: {
        'Content-Type': 'application/json',
        Authorization: `Bearer ${config.token}`,
      },
      body: JSON.stringify({
        streams: events.map(e => ({
          stream: { service: e.service, level: e.level },
          values: [[String(Date.parse(e.timestamp) * 1e6), JSON.stringify(e)]],
        })),
      }),
    }),
  })
}
```

`defineHttpDrain` handles abort timeouts, exponential backoff on `5xx` / network errors, and automatic identity headers — you only describe how to encode events into the request.
