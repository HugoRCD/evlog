Practical patterns for using evlog with NestJS. The `evlog/nestjs` middleware auto-creates a request-scoped logger accessible via `req.log` and `useLogger()`, emitting a wide event when the response completes.

## Setup

### 1. Install dependencies

```bash
npm install evlog express
```

### 2. Initialize and apply middleware

```typescript [src/main.ts]
import express from 'express'
import { initLogger } from 'evlog'
import { evlog } from 'evlog/nestjs'

initLogger({
  env: { service: 'my-api' },
})

const app = express()
app.use(evlog())

app.get('/health', (req, res) => {
  req.log.set({ route: 'health' })
  res.json({ ok: true })
})

app.listen(3000)
```

`req.log` is the evlog wide-event logger attached to the request. The middleware uses Express-compatible `app.use()` which works with NestJS's default Express adapter.

## Wide Events

Build up context progressively through your handler. One request = one wide event:

```typescript [src/routes/users.ts]
app.get('/users/:id', async (req, res) => {
  const { id } = req.params

  req.log.set({ user: { id } })

  const user = await db.findUser(id)
  req.log.set({ user: { name: user.name, plan: user.plan } })

  const orders = await db.findOrders(id)
  req.log.set({ orders: { count: orders.length, totalRevenue: sum(orders) } })

  res.json({ user, orders })
})
```

All fields are merged into a single wide event emitted when the request completes:

```bash [Terminal output]
14:58:15 INFO [my-api] GET /users/usr_123 200 in 12ms
  ├─ orders: count=2 totalRevenue=6298
  ├─ user: id=usr_123 name=Alice plan=pro
  └─ requestId: 4a8ff3a8-...
```

## useLogger()

Use `useLogger()` to access the request-scoped logger from anywhere in the call stack — no need to pass the request object through your service layer:

```typescript [src/services/user.ts]
import { useLogger } from 'evlog/nestjs'

export async function findUser(id: string) {
  const log = useLogger()
  log.set({ user: { id } })

  const user = await db.findUser(id)
  log.set({ user: { name: user.name, plan: user.plan } })

  return user
}
```

```typescript [src/main.ts]
import { findUser } from './services/user'

app.get('/users/:id', async (req, res) => {
  const { id } = req.params
  const user = await findUser(id)
  res.json(user)
})
```

Both `req.log` and `useLogger()` return the same logger instance. `useLogger()` uses `AsyncLocalStorage` to propagate the logger across async boundaries.

## Error Handling

Use `createError` for structured errors with `why`, `fix`, and `link` fields:

```typescript [src/main.ts]
import { createError, parseError } from 'evlog'

app.get('/checkout', async (req, res, next) => {
  try {
    throw createError({
      message: 'Payment failed',
      status: 402,
      why: 'Card declined by issuer',
      fix: 'Try a different payment method',
      link: 'https://docs.example.com/payments/declined',
    })
  } catch (error) {
    req.log.error(error as Error)
    const parsed = parseError(error)
    res.status(parsed.status).json({
      message: parsed.message,
      why: parsed.why,
      fix: parsed.fix,
      link: parsed.link,
    })
  }
})
```

The error is captured and logged with both the custom context and structured error fields:

```bash [Terminal output]
14:58:20 ERROR [my-api] GET /checkout 402 in 3ms
  ├─ error: name=EvlogError message=Payment failed status=402
  └─ requestId: 880a50ac-...
```

## Drain & Enrichers

Configure drain adapters and enrichers directly in the middleware options:

```typescript [src/main.ts]
import { createAxiomDrain } from 'evlog/axiom'
import { createUserAgentEnricher } from 'evlog/enrichers'

const userAgent = createUserAgentEnricher()

app.use(evlog({
  drain: createAxiomDrain(),
  enrich: (ctx) => {
    userAgent(ctx)
    ctx.event.region = process.env.FLY_REGION
  },
}))
```

### Pipeline (Batching & Retry)

For production, wrap your adapter with `createDrainPipeline` to batch events and retry on failure:

```typescript [src/main.ts]
import type { DrainContext } from 'evlog'
import { createAxiomDrain } from 'evlog/axiom'
import { createDrainPipeline } from 'evlog/pipeline'

const pipeline = createDrainPipeline<DrainContext>({
  batch: { size: 50, intervalMs: 5000 },
  retry: { maxAttempts: 3 },
})
const drain = pipeline(createAxiomDrain())

app.use(evlog({ drain }))
```

::callout{icon="i-lucide-info" color="info"}
Call `drain.flush()` on server shutdown to ensure all buffered events are sent. See the [Pipeline docs](/adapters/pipeline) for all options.
::

## Tail Sampling

Use `keep` to force-retain specific events regardless of head sampling:

```typescript [src/main.ts]
app.use(evlog({
  drain: createAxiomDrain(),
  keep: (ctx) => {
    if (ctx.duration && ctx.duration > 2000) ctx.shouldKeep = true
  },
}))
```

## Route Filtering

Control which routes are logged with `include` and `exclude` patterns:

```typescript [src/main.ts]
app.use(evlog({
  include: ['/api/**'],
  exclude: ['/_internal/**', '/health'],
  routes: {
    '/api/auth/**': { service: 'auth-service' },
    '/api/payment/**': { service: 'payment-service' },
  },
}))
```

bun install
bun run example:nestjs
